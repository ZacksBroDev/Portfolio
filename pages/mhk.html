Mile High Karate Training Platform

Full-Stack React Application for Martial Arts Education

üéØ The Problem

Mile High Karate needed to digitize their belt-level curriculum and provide students with 24/7 access to training materials. The existing solution was scattered PDFs and YouTube videos with no access control, meaning students could access content beyond their belt level, undermining the dojo's teaching methodology.

Key challenges:

  • Implement secure, belt-specific content gating without overly complex infrastructure
  • Support 100+ concurrent users on a limited budget (sub-$50/month hosting)
  • Enable instructors to update content without developer intervention
  • Maintain fast load times for video-heavy content

üë• Users & Use Cases

Primary Users: Students (ages 6-60) accessing belt-specific training videos, written materials, and belt requirements.

Secondary Users: Instructors updating curriculum, tracking student progress, and managing content.

Core User Flow:

  1. Student logs in via Auth0 (Google/email)
  2. System reads user's belt level from MongoDB profile
  3. React front-end renders only authorized content
  4. Student streams videos, downloads PDFs, views technique breakdowns

üèóÔ∏è Architecture & Tech Decisions

Tech Stack

  • Front-End: React 18 with React Router for SPA navigation
  • Back-End: Node.js + Express API on AWS EC2
  • Database: MongoDB Atlas (free tier, ~2GB data)
  • Auth: Auth0 for authentication + JWT validation
  • Storage: AWS S3 for video/PDF storage with CloudFront CDN
  • Hosting: Vercel for React front-end, EC2 t3.micro for API

Why These Choices?

Auth0 over roll-your-own: Security is critical when dealing with minors. Auth0 provides SOC 2 compliance, MFA, and saves ~200 hours of security implementation/maintenance.

MongoDB over PostgreSQL: Belt levels and curriculum are document-oriented (nested arrays of techniques, videos, requirements). MongoDB's flexible schema allowed rapid iteration during curriculum changes.

S3 + CloudFront over direct streaming: Offloads 95% of bandwidth to AWS CDN. Without this, video streaming would cost $200+/month. CloudFront reduces latency from ~2s to ~300ms for video start.

React SPA over SSR: No SEO requirements (members-only site). SPA provides instant navigation and offline-capable PWA potential for future iterations.

‚öñÔ∏è Tradeoffs & Lessons Learned

What Worked

  • Auth0 integration: Zero security incidents, 99.9% uptime. Users can self-service password resets.
  • CloudFront CDN: Reduced video buffering complaints by 90% vs. direct S3.
  • MongoDB flexible schema: Allowed curriculum restructuring 3x without migrations.

What I'd Change

  • Over-reliance on client-side filtering: Belt-level gating happens in React. A determined user could inspect network calls and access unauthorized videos. Would refactor to enforce access control at API level with signed S3 URLs that expire.
  • No automated testing: Launched without Jest/Cypress. Instructor content updates occasionally break student views. Would implement E2E tests covering auth flows and content rendering before next release.
  • Manual deployment: Currently SSH + git pull + pm2 restart. Should implement CI/CD with GitHub Actions for automated testing and zero-downtime deploys.
  • Video encoding: All videos are uploaded raw. Should implement AWS MediaConvert for adaptive bitrate streaming (HLS/DASH) to support low-bandwidth users.
  • Analytics gap: No visibility into which videos get watched most, bounce rates, or session lengths. Would integrate PostHog or similar to inform curriculum prioritization.
